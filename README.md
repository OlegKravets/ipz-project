# Проект по предмету "Инженерия программного обеспечения"

### Описание спортзалов.
```
В наличии имеется несколько тренажерных залов (от двух и более). В одном зале есть основная часть и комната для 
групповых занятий. В каждом из залов размеры могут варьироваться.
```

### Основная часть зала.
```
В основной части работают несколько тренеров. (приводить своих тренеров запрещено) Каждый тренер имеет 
определенный уровень квалификации. Также в основной части зала имеются кардио зоны, тренажеры и другие 
приспособления.
```

### Описание комнат для групповых занятий.
```
Каждая комната имеет своё назначение. В ней есть своё специализированное оборудование и тренер имеющий 
соответствующую квалификациию. Администратор должен следить за занятостью комнат (если определенный вид 
занятий популярнее, к примеру, выделять дополнительную комнату в то же время).
```

### Reception
```
Является рабочим местом администратора, который регистрирует новых клиентов, выдает им персональные 
карточки. Также имеется мини магазин (с водой и спортивным питанием).
```

### Услуги доступные клиенту
```
Клиент может зарегистрироваться в системе (что не обязательно, возможно просто посещать по индивитуальной 
карточке). Может записаться как на групповое занятие, так и на посещение основной части зала. С помощью клиентского 
приложения возможно записываться  на занятия любого вида, оплачивать абонемент, просматривать занятость залов, а 
также информацию о тренере. После записи на занятие пользователю приходит уведомление о скором начале занятия. 
Уведомления о скидках (отпревляются администратором). Реклама, при появлении новых типов занятий.
```

### Функции доступные тренеру.
```
Тренер может зарегистрироваться (должен быть обязательно верифицирован, для чего должен связываться с администратором). 
Имеет возможность общения с клиентами. Есть список заявок на тренировки (приходящие от клиентов).
```
